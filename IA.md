# ontoMeta 信息架构文档

## 1. 信息架构目标

信息架构的目标，是把 ontoMeta 的核心能力围绕本体组织起来，使用户能以最短路径完成以下任务：

- 进入某个数据域开始建模
- 查看本体草稿与生成结果
- 理解业务对象、属性、关系和业务逻辑
- 在工作区维护语义模型

---

## 2. 一级导航

### 2.1 工作区

面向建模过程，按 DataHub 数据域组织。

#### 页面职责

- 提供本体草稿预处理入口
- 预处理时由后台自动拉取对应 DataHub 元数据
- 提供草稿结果编辑入口
- 提供继续处理与保存入口
- 提供 DataHub 页面外链作为可选查看入口

#### 核心对象

- Domain
- 预处理任务
- 草稿结果
- 变更状态

### 2.2 本体

面向语义结果，是系统核心页面。

#### 页面职责

- 展示业务对象
- 展示业务对象之间的关系
- 进入对象详情查看完整本体信息
- 仅提供只读浏览，不提供编辑动作

#### 核心对象

- Object Type
- Property
- Relation Type
- Business Logic
- Version

### 2.3 业务逻辑

面向规则、指标、标签等逻辑资产。

#### 页面职责

- 展示业务逻辑列表
- 查看逻辑详情
- 展示逻辑依赖的对象、字段、表与规则来源
- 仅提供只读浏览，不提供编辑动作

#### 核心对象

- Business Logic
- Logic Source
- Related Object Types
- Related Datasets

### 2.4 智能问数（Chat BI）

面向已发布本体的自然语言问答。

#### 页面职责

- 会话列表与分类管理
- 提问、回答与本体引用展示
- 无检索命中时明确拒绝编造

### 2.5 外部 API

面向集成方与运维的对外能力控制台。

#### 页面职责

- 创建/管理外部应用与 API Key（仅创建或轮换时可见明文）
- 浏览 MCP / REST 目录并试用（真实鉴权）
- 查看应用级 scope 与调用约束说明

### 2.6 设置

- 管理鉴权 Token 配置（前端本地）
- LLM 服务与 DataHub 连接配置

## 3. 页面层级

## 3.1 工作区

### 3.1.1 数据域列表页

展示字段：

- 数据域名称
- 数据域说明
- 草稿数量
- 已发布数量
- 最近更新时间

可执行动作：

- 进入数据域详情
- 发起本体草稿生成
- 查看建模进度

### 3.1.2 数据域详情页

模块组成：

- 数据域概要
- 本体草稿概览
- 预处理任务状态
- 最近变更概览
- DataHub 外链入口

可执行动作：

- 生成草稿
- 查看草稿详情
- 编辑草稿结果
- 确认重要操作
- 进入本体页
- 进入业务逻辑页

---

## 3.2 本体

### 3.2.1 本体列表页

展示方式：

- 列表视图
- 卡片视图
- 图谱视图

筛选条件：

- 数据域
- 状态
- 对象类型
- 标签
- 所属团队

列表字段：

- 对象名称
- 对象类型
- 属性数量
- 关系数量
- 业务逻辑数量
- 状态
- 最近更新时间

### 3.2.2 本体对象详情页

模块组成：

- 基本信息
- 来源信息
- 属性列表
- 关系列表
- 业务逻辑列表
- 关联数据集
- 版本记录

可执行动作：

- 查看版本差异
- 跳转到工作区编辑

### 3.2.3 关系图谱页

展示内容：

- 对象节点
- 关系边
- 对象分组
- 跨域关系

交互动作：

- 点击节点查看详情
- 按数据域筛选
- 按关系类型筛选
- 查看上下游邻接对象

---

## 3.3 业务逻辑

### 3.3.1 业务逻辑列表页

展示字段：

- 逻辑名称
- 逻辑类型
- 所属数据域
- 关联对象数量
- 来源类型
- 状态
- 最近更新时间

筛选条件：

- 逻辑类型
- 来源类型
- 数据域
- 状态

### 3.3.2 业务逻辑详情页

模块组成：

- 基本信息
- 逻辑定义
- 计算规则 / 标签规则
- 绑定对象（含 role：主对象 / 维度对象 / 产出对象）
- 绑定字段（含 role：输入 / 输出 / 过滤 / 分组）
- 来源 SQL / 代码 / 人工补充
- 相关图表 / 看板
- 版本记录

可执行动作：

- 编辑逻辑定义
- 新增 / 解除对象绑定、字段绑定（在工作区，业务逻辑页只读）
- 修改绑定 role
- 修改逻辑类型
- 发布版本

---

## 4. 全局能力

### 4.1 全局搜索

支持搜索：

- 数据域
- 业务对象
- 属性
- 关系
- 业务逻辑
- 数据集
- 术语

### 4.2 全局筛选

- 数据域
- 状态
- 标签
- 所属团队
- 更新时间

### 4.3 版本与审计

所有核心对象均应支持：

- 当前状态展示
- 历史版本查看
- 变更对比
- 审核与发布记录

---

## 5. 关键导航路径

### 5.1 建模路径

工作区 -> 数据域详情 -> 生成草稿 -> 草稿审核 -> 发布 -> 本体详情

### 5.2 理解路径

本体列表 -> 本体详情 -> 关系列表 / 图谱 -> 业务逻辑详情

### 5.3 追溯路径

业务逻辑详情 -> 关联对象 -> 来源数据集 -> DataHub 元数据来源

---

## 6. 架构原则

- 一级导航以「本体 / 工作区 / 业务逻辑」为核心，并扩展「智能问数 / 外部 API / 设置」
- 工作区是过程视角，本体是结果视角，业务逻辑是规则视角
- 本体详情页是最关键的信息聚合页
- 所有页面都应支持从语义回到来源，从来源回到语义
- 鉴权：管理面用 Admin Token；对外集成用 App Key（详见 PRD / TECH_DESIGN）
